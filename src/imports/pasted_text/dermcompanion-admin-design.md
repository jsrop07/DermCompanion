Design a high-quality, modern, premium healthcare admin web dashboard for a dermatology clinic service called DermCompanion.

DermCompanion is a post-procedure patient care and medication tracking service. In this system, clinic staff such as doctors, nurses, coordinators, and administrators use a web dashboard to manage patients after dermatology procedures. This design is for the admin web only.

At this stage, do NOT include any AI features. Do not create photo upload review, AI triage, AI chatbot, or any automated diagnosis interface. Focus only on the pre-AI MVP workflow.

The web dashboard should feel professional, efficient, structured, premium, medically trustworthy, and easy to scan quickly in a real clinic environment. It should look like a polished healthcare SaaS dashboard, not a generic template. The design should support real operational work by clinic staff.

Use a clean desktop-first responsive web layout with:
- left sidebar navigation
- top header area
- summary cards
- data tables
- patient detail panels
- status badges
- filter bars
- tabs where needed
- modal design where appropriate
- card + table hybrid layouts

Visual style:
- premium healthcare SaaS
- clean white and very light gray base
- calm blue, teal, mint accents
- subtle shadows
- rounded corners
- soft borders
- high readability
- excellent spacing
- modern but not overly trendy
- professional and clinical, but not cold
- slightly premium private dermatology clinic feel

Avoid:
- playful illustrations
- childish icons
- dark mode
- crowded screens
- gaming UI
- AI assistant panels
- futuristic sci-fi visuals
- overly colorful dashboards

Important product context:
This admin dashboard is used after a patient has received a dermatology treatment or procedure. Staff use the dashboard to register patients, configure their post-procedure care flow, manage medication schedules, monitor recovery progress, and respond to appointment-related requests.

Create the following desktop web screens in a consistent design system:

1. Admin Login Screen
Design a premium, trustworthy clinic admin login page.
Include:
- DermCompanion logo area
- page title
- admin ID or email input
- password input
- login button
- subtle supporting text for clinic staff
- optionally a side branding panel or clean visual section
The page should feel secure, simple, and polished.

2. Admin Main Dashboard
This is the main homepage for clinic staff.
Create a dashboard that quickly communicates the current clinic operational status.
Include:
- top header with page title, clinic name, notification icon, profile/avatar area
- summary cards for:
  - today’s registered patients
  - active recovery patients
  - patients with missed medication logs
  - appointment change requests
- recent patients table
- quick action section
- recent requests / alerts panel
- clear status badges such as normal, pending, missed, in progress
- compact data visualization only if useful, but keep it lightweight and practical
The layout should be easy to scan in a few seconds.

3. Patient Registration Screen
Design a patient registration and setup page used by clinic staff.
Include a structured form layout with sections such as:
- patient basic information
  - patient name
  - phone number
  - birth date
- treatment / procedure information
  - procedure name
  - procedure date
  - procedure time
- medication setup
  - medication name
  - dosage
  - frequency
  - purpose
- recovery guide template selection
- appointment information if needed
- action buttons:
  - register patient
  - send SMS link
Use form cards or grouped panels so the screen feels organized and premium, not like a plain old admin form.

4. Patient List Screen
Design a full patient management screen with a searchable and filterable table.
Include:
- page title
- filter bar
- search input
- status filters
- date filter
- patient table with columns such as:
  - patient name
  - procedure
  - procedure date
  - recovery stage
  - medication status
  - appointment status
  - last updated
- action buttons in each row:
  - view details
  - edit
  - resend link
- empty state design and pagination if helpful
The table should feel realistic and clinic-friendly.

5. Patient Detail Screen
This is one of the most important screens.
Design a detailed patient profile and tracking page that allows staff to view the patient’s current post-procedure progress.
Include:
- patient header section with patient name, basic info, current status badge
- procedure summary card
- current recovery progress card
- medication list section
- medication adherence history section
- appointment information section
- internal notes section for staff
- timeline or status area showing current recovery stage
- action buttons such as:
  - edit medication info
  - edit recovery guide
  - save note
  - resend patient access link
Use tabs if useful, for example:
- overview
- medication
- appointments
- notes
This screen should feel information-rich but well organized.

6. Appointment Request Management Screen
Design an admin screen where staff review patient appointment-related requests.
Include:
- list or table of appointment change requests
- columns such as:
  - patient name
  - current appointment
  - requested change
  - request date/time
  - request status
- row actions:
  - review
  - mark completed
  - contact patient
- request detail panel or side drawer if useful
The screen should support fast administrative handling.

7. Clinic Information Management Screen
Design a settings-style admin page for clinic profile information.
Include:
- clinic name
- phone number
- address
- business hours
- notice / guidance text
- save button
- optional preview card of patient-facing clinic info
The page should feel simple, neat, and editable.

8. Recovery Guide Management Screen
Design a content management page where clinic staff can configure post-procedure recovery instructions.
Include:
- recovery guide template list
- categorized sections by time period
- examples such as 3 hours, 12 hours, 24 hours, 48 hours, 72 hours
- structured editor blocks or form sections
- save/update controls
- template organization
This should feel like a professional healthcare content management screen, not a generic document editor.

9. Optional Admin Navigation Structure
Use a left sidebar with these menu items:
- Dashboard
- Patients
- Patient Registration
- Medication Tracking
- Appointment Requests
- Recovery Guide Management
- Clinic Information
- Settings
Highlight the current page clearly.
